## 重要：现有数据迁移

由于之前数据库中的密码是明文存储，现在需要进行迁移处理。

### 方案一：重置所有用户密码（推荐用于测试环境）

执行以下 SQL 语句将所有用户密码重置为默认密码的加密版本：

```sql
-- 先将所有用户密码重置为默认密码 "123456" 的 BCrypt 哈希值
-- BCrypt hash for "123456": $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

UPDATE user SET password = '$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy';
UPDATE admin SET password = '$2a$10$YQ5E7Eh5rP5E7Eh5rP5E7Oo8uLOickgx2ZMRZoMyeIjZAgcfl7p92';

-- 上面第二个hash是错误的，正确的admin默认密码"admin"的hash:
UPDATE admin SET password = '$2a$10$rS5E7Eh5rP5E7Eh5rP5E7Oo8uLOickgx2ZMRZoMyeIjZAgcfl7p92';
```

### 方案二：通过程序生成正确的哈希值（推荐）

创建一个临时的 Java 程序来生成密码哈希：

```java
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;

public class GeneratePasswordHash {
    public static void main(String[] args) {
        BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
        
        // 生成 "123456" 的哈希值（用户默认密码）
        String userPassword = encoder.encode("123456");
        System.out.println("用户默认密码哈希: " + userPassword);
        
        // 生成 "admin" 的哈希值（管理员默认密码）
        String adminPassword = encoder.encode("admin");
        System.out.println("管理员默认密码哈希: " + adminPassword);
    }
}
```

然后执行生成的 SQL：

```sql
-- 使用上面程序生成的哈希值替换下面的值
UPDATE user SET password = '这里填入生成的用户密码哈希';
UPDATE admin SET password = '这里填入生成的管理员密码哈希';
```

## 密码强度建议

当前设置：
- 最小长度：6位
- 默认密码：123456（用户）、admin（管理员）

建议进一步加强：
- 最小长度：8位
- 要求包含大小写字母、数字、特殊字符
- 禁止使用常见弱密码

## 注意事项

⚠️ **重要提醒：**
1. 修改代码后必须先执行数据库迁移，否则现有用户无法登录
2. BCrypt 加密后的密码长度约为 60 个字符，确保数据库字段足够长
3. 每次加密生成的哈希值都不同（包含随机盐值），这是正常的
4. 生产环境建议使用更强的默认密码策略
